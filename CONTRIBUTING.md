# Contributing Guidelines

We welcome contributions that align with the project's goals of reproducibility, compliance, and operational clarity.

## 🛠️ How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Commit your changes with clear messages
4. Ensure all tests pass (`./mvnw test`)
5. Submit a pull request with context and rationale

## ✅ Requirements

- Code must be compatible with JDK 21 and Spring Boot 3.5.5
- Avoid deprecated APIs and ensure memory safety
- Include JUnit tests for new features
- Update documentation if behavior changes

## 📞 Communication

For major changes, open an issue first to discuss your proposal.  
For questions, reach out via [jessereyj@linux.com](mailto:jessereyj@linux.com)
